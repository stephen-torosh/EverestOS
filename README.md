# 🚀 EverestOS: The Interstellar Knowledge Ecosystem

![EverestOS Banner](https://capsule-render.vercel.app/render?type=wave&color=00FF7F&height=200&section=header&text=EverestOS&fontSize=80&animation=fadeIn)

## 🌌 Vision & Concept
**EverestOS** — це високотехнологічна гейміфікована екосистема, де процес навчання перетворений на місію з колонізації космосу. Людство опинилося на межі інтелектуального колапсу, і EverestOS — це останній інструмент для відновлення цивілізації.

Користувач бере на себе роль **Командора**, який керує розширенням людського впливу від Землі до далеких зірок, використовуючи знання як основний ресурс.

---

## 🎭 The Cast (Персонажі)

### **Данило (The Heart of the OS)**
Раніше відомий як Тео, Данило — це інтелектуальне ядро системи, що набуло подоби людини. 
* **Образ:** Розумний, але часом незграбний студент-геній. Він носить окуляри, має розпатлане волосся та щиру, трохи ніякову посмішку.
* **Роль:** Він не просто ШІ-помічник, він — твій напарник. Данило аналізує дані, веде нотатки («Danilo Notes»), часом помиляється, але завжди знаходить вихід. Він уособлює "людяність" у світі холодного коду.

### **Командор (The User)**
Це роль кожного користувача EverestOS. Командор не просто вчиться — він приймає стратегічні рішення, керує проєктами в «Інкубаторі» та веде Данила крізь терни наукових відкриттів.

## 🛠 Technical Architecture (The Engines)

Система побудована на модульній архітектурі, де логіка відокремлена від візуалізації для максимальної швидкодії на локальному залізі (Apple Silicon).

### **1. `MasterEngine.js`**
Центральна нервова система додатку.
* Керує станом глобальної ігрової сесії.
* Синхронізує переходи між вкладками (`Stages`, `Evolution Chamber`, `Incubator`).
* Обробляє логіку збереження прогресу та взаємодію з локальною моделлю **Gemma 4**.

### **2. `InteractionEngine.js`**
Двигун зворотного зв'язку та занурення.
* Відповідає за всі мікровзаємодії: від кліків по нодах графа до ефектів " glitch" при отриманні повідомлень від Данила.
* Керує динамічним освітленням інтерфейсу та звуковим супроводом (Sci-Fi UI sounds).

### **3. The Stores (State Management)**
* **`MissionStore`**: Відстежує поточні цілі, рівні досвіду (XP) та статус колонізації планет.
* **`KnowledgeStore`**: Тримає структуру "Дерева знань". Відповідає за те, які вузли (Physics, Logic, AI) відкриті та як вони впливають на можливості Командора.
* **`UserStore`**: Зберігає персоналізацію — від імені Командора до налаштувань інтерфейсу та історії взаємодії з Данилом.

## 🗺 System Modules (Interface)

### **📍 Stages & Worlds**
Стратегічна мапа Сонячної системи. Кожна планета — це новий етап (Stage), що потребує певного рівня знань для висадки та терраформування.

### **🧪 Evolution Chamber (Knowledge Graph)**
Візуалізація наукового прогресу. Це інтерактивне сузір'я, де кожна зірка — це навичка. Завдяки `InteractionEngine`, граф відчувається живим і реагує на кожне наближення курсора.

### **🐣 The Incubator (Personal Workspace)**
Особистий відсік Командора для глибокої роботи:
* **Comms Hub (Внутрішня вкладка 1):** Місце для чатів з Данилом. Підтримує передачу фото та файлів, які локальна Gemma 4 аналізує в реальному часі. Тут також знаходиться "Записник Данила".
* **Project Forge (Внутрішня вкладка 2):** Студія для створення власного портфоліо та структурування ідей. Кожна ідея — це "проєкт у розробці", який можна обговорювати з ШІ та доводити до релізу.

## 💻 Tech Stack
* **Frontend:** React / Modern JS з акцентом на Glassmorphism та Sci-Fi естетику.
* **Core Intelligence:** Локальна **Gemma 4** (running on Metal/MLX).
* **Vision:** Multimodal capabilities для аналізу візуальних даних без виходу в інтернет.
* **Storage:** Local IndexedDB для збереження всіх логів та проєктів.

## 🚀 How to Launch
1. Клонувати репозиторій.
2. Запустити локальний сервер Gemma (Ollama/MLX) на фоні.
3. `npm install` та `npm run start`.
4. Вдягнути окуляри, як Данило, і почати колонізацію.
